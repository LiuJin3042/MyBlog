    配置DHCP服务器

    基本配置

    • 排除地址

      ip dhcp 192.168.0.1 192.168.0.10 // 排除一个地址段
ip dhcp 192.168.0.254 // 排除一个地址

    • 配置地址池

      ip dhcp pool Name-of-your-pool
(dhcp-config)
network 192.168.0.0 255.255.255.0
default-router 192.168.0.1
dns-server 192.168.11.5
domain-name cisco.com
end

    • 禁用/启用

      no service dhcp
service dhcp

    验证

    show run | section dhcp
show ip dhcp bingding
show ip dhcp server statistics

    DHCPv4中继

    只需要部署一台DHCP服务器, 整个网络都可以传播DHCP请求. 解决DHCP广播不能跨路由的问题.

    在路由器上设置

    inter g0/0
ip helper-address 192.168.11.6 // 在g0/0收到dhcp请求, 会单播发给192.168.11.6

    配置DHCPv4客户端

    小型办公室/家庭办公室(SOHO)中的路由器的互联网端口常常配置为DHCP, 接受运营商的IP地址. 

    inter g0/1
ip addr dhcp
no shut

      show ip dhcp conflict

    DHCPv6

    无状态地址自动配置(SLAAC)

    在没有 DHCPv6 服务器服务的情况下获取 IPv6 全局单播地址

    • 路由器请求(RS)

      客户端使用SLAAC获取编址信息, 把RS消息发给路由器, 使用组播地址FF02::2

    • 路由器通告(RA)

      RA 消息包括本地数据段的前缀和前缀长度. 定期发送RA.使用组播地址FF02::1

      后缀通过随机的方式生成, 或使用EUI-64通过MAC地址生成.

      在使用这个地址前, 会发送ICMPv6的邻居请求消息. 发送后24位. 如果没有收到应答则采用. 称为邻居地址检测(DAD)

    仅SLAAC

    通告前缀、前缀长度、DNS 服务器、MTU 和默认网关信息

    将M标记O标记都设置为0

    no ipv6 nd managed-config-flag
no ipv6 nd other-config-flag

    无状态DHCPv6

    客户端使用 RA 消息中的前缀和前缀长度

    与无状态 DHCPv6 服务器通信以获取 RA 消息中未提供的其他信息. 例如, 这可能是 DNS 服务器 IPv6 地址列表. 

    ipv6 nd other-config-flag // O设为1. M默认是0

    有状态DHCPv6

    不用RA, 从DHCPv6服务器获取所有信息

    ipv6 nd managed-config-flag // M设为1. O默认是0

ipv6 dhcp poop NAME-of-POOL
address prefix .../64 lifetime infinite
dns server [dns服务器地址]
domain-name example.com
inter g0/0
ipv6 addr ...
ipv6 dhcp server NAME-of-POOL

debug ipv6 dhcp detail
